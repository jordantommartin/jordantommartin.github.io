Custom C++ Engine

Programming 2 Individual Project / C++ and OpenGL

Class Description
Templated Singly Linked List
  • A templated Singly Linked List that uses nodes to hold a piece of data and point to the next node in the list.
  • Provides custom implementations of copy and move semantics.
  • Has a custom iterator that allows nodes to be incremented, compared, and dereferenced.
Templated Vector
  • A templated Vector that an array pointer to hold data.
  • Provides custom implementations of copy and move semantics.
  • Has a custom iterator that allows elements at each index to be incremented, added/substracted, compared, and dereferenced.
Templated HashMap
  • A templated HashMap that contains a Vector filled with SLists that contains Key, Value pairs.
  • Includes a hash function object for hashing keys and a equality function object for comparing values.
Datum
  • Data storage container for an array of values that can be defined at runtime.
  • Data types can be either primitve or user-defined.
  • Introduces custom Runtime Type Identification data type which allows check an object's type before casting it.
Scope
  • Uses a hashmap to create pairs of strings and Datums, known as a table.
  • Each Datum can contain pointers to further Scopes, thereby forming a hierarchical data structure.
  • Each nested scope has a pointer to its parent scope, allowing search down or up the hierarchy.
Attributed Scope
  • A pure virtual class that inherits from the Scope class and adds the ability to populate Scopes using schema at compile time.
  • Attributes allow us to mirror C++ classes within scopes, with the added ability of allowing us to set data members of these classes during runtime.
  • A type manager is introduced which registers any class that inherits from an Attributed Scope.
Parse Coordinator and Table Parser
  • A parser that translates JSON files into a configuration that can be used to populate Attributed Scopes.
  • Implements chain of responsibility design pattern when parsing, which allows for different kinds of JSON grammars to be read.
  • The parse coordinator is given a JSON file which searches through its list of parse helpers to find one that will handle this type of grammar.
  • Once a parse helper is capable, the JSON configuration is read. If no helper if capable, the request is not handled.
Factory
  • Templated abstract base class used to create concrete objects from their default contructor.
  • Derived factories for each type of class are generated through a macro and stored in a FactoryManager, which can be used to instantiate objects using the class name.
  • Game engine will receive the class name from the parse JSON configuration and instantiate objects of that class at runtime.
Entity
  • A wrapper for an Attributed Scope that populates its tables at runtime using the Factory and JSON configuration.
  • Similar to Unity's concept of a GameObject, which can contain further GameObjects within itself.
  • Contains a virtual update method which can be implemented in derived classes for game logic.
Action
  • Abstract base class that extends from attributed scope and used to modify entities.
  • Actions are defined in JSON configuration and attached in the hierarchy underneath an entities.
  • Contains a virtual update method which can be implemented in derived classes for game logic.
  • ActionLists allow for multiple actions to be invoked, similar to a functions for C++ classes.
  • Actions created include: ActionIf, ActionIncrement, ActionExpression, ActionCreateAction, ActionDestroyAction
Event
  • Event system that implements the Observer design pattern. Allows us to skip polling every action on the Update methods, which is expensive.
  • Any class that derives from the EventSubcriber class can subscribe to an EventPublisher. Calling the Deliver method in the publisher will then call the notify method in the subscriber.
  • Events carry a templated data "payload" and can be delivered immediately or queued for future delivery.
Reaction
  • Calls an ActionList in response to an event delivery. Intended to act as a function call.
  • Allows for us to use the payload from the event as a parameter to the Actions that are called in the ActionList.

Super Paul Box

Super Paul Box was a team project between myself and 4 other programmers built on top of the Custom C++ Engine. Specifically, I created both the Audio Manager and Score Keeping system. This was accomplished by utilizing an event queue in our engine to subscribe to Audio or Score events.
