Zenko: A Fox's Tale

Capstone Project / Unreal Engine

Zenko: A Fox's Tale is a 3D Action-Plaformer being developed by Wolf Packtics, an interdisciplinary team of 17 people including programmers, artists, and producers. The game is about a swift fox named Zenko that can jump, pounce, and dash his way through old ruins to chase down spirits to rebuild a broken shrine. Zenko can also swap realms to find hidden platforms and secrets throughout the levels. The game was made for players who love to explore and was inspired by games like Ori and the Blind Forest and Spirit of the North.

Custom Destructible Mesh Objects

One of the ideas we had discussed when we created Zenko was the idea that the physical world would be destroyed, while the spirit realm represents the past where structures were still intact. I was tasked with making this a reality and started working with destructible meshes to manifest this idea.

Using the Apex destruction plugin, I was able to manipulate destructible meshes in a way that we could "rewind" the destruction. The object uses a combination of destructible meshes, poseable meshes, and static meshes to cycle between different states that dictate which mesh to use and which transformations to apply to the pieces.

Once I had this in place, I was able to connect this object to our spirit realm swap mechanic so that when Zenko switches realms, the object will build or destroy itself. We have plans to create obstacles from these objects and create a shrine that can be rebuilt throughout the game.

Movement Controller

Zenko's movement controller was our first major task and I was able to contribute by working on his abilities in the game. These included his dash and his pounce. I created the dash by taking Zenko's forward vector and normalizing it. Then multiplying this by a certain amount to dash forward. I improved upon the pounce by adding a bounce back as well as a particle effect for when he touches the ground.

GameAnalytics

I also integrated GameAnalytics API with our project to record important metrics, such as the amount of time a section of the level took or how many secret shrines a player was able to find. I was able to do this by creating an AnalyticsManager blueprint with functions for sending API requests on behalf of each situation we wanted to record.
